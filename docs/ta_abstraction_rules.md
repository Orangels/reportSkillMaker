# TA 抽象策略规则

模板分析的核心原则：输出的是**可复用的生成规则**，不是当期模板的数据快照。

## 策略一：实例消除与参数化

将模板中随期变化的具体数据值替换为占位符。

**判断标准**：这个值在下一期报告中是否还会相同？不同→替换为占位符。

| 替换前 | 替换后 |
|--------|--------|
| 11月份 | XX月份 |
| 47起 | N起 |
| 治安殴打他人 | [警情类型A] |
| 环比上升17.5% | 环比[上升/下降]X% |
| 西门所、新盈所 | [高发辖区列表] |

**适用场景**：结构框架中的章节标题、排列顺序的排序示例、句式模板中的例句。

## 策略二：从实例集合到生成规则

将穷举的具体映射关系压缩为一条生成规则 + 结构模板。

| 替换前 | 替换后 |
|--------|--------|
| 10行问题-建议对应表 | "工作建议与前文分析的问题一一对应" + 建议结构模板（责任主体、具体措施、量化要求、预期效果） |
| 8个派出所名称列表 | "辖区单位名称列表（从数据源获取）" |
| 穷举所有加粗词汇 | "对分类名称、分析维度标题、小结标记、要点序号使用段内加粗" |

**适用场景**：工作建议对应关系、专业术语枚举、加粗对象列表。

## 策略三：按稳定性分层处理

对"什么该具体、什么该抽象"的判断标准：看这个值是否**跨期稳定**。

| 稳定性 | 处理方式 | 示例 |
|--------|---------|------|
| 固定不变 | 保留具体值 | 六大警情类别名称、格式规范参数（字体/字号/行距）、发文单位名称 |
| 随数据变化 | 抽象为参数 | 重点分析的警情类型、具体数量和占比、高发辖区名称 |
| 结构随数据变化 | 描述变化规则 | 二级章节数量、分析维度选择、工作建议条数 |

**适用场景**：数据指标体系、总分验证关系、可变元素分析。

## 策略四：功能导向重组

组织信息的视角从"模板里有什么"转为"写报告时需要什么"。

| 替换前（信息提取视角） | 替换后（写作指导视角） |
|----------------------|---------------------|
| 术语按领域分类（警情分类、辖区单位） | 术语按功能分类（动作描述、趋势描述、评价用语） |
| 层级关系标注类型（总分总、并列、递进、因果） | 过程指令（数据呈现→特征归纳→风险研判→对策建议） |
| "三级章节之间为递进关系" | "每个分析维度按数量降序排列，用'主要集中在...其次...'句式" |

**适用场景**：专业术语表、论证逻辑链、段落组织模式。

## 策略五：归一化合并

当多种模式的本质结构相同时，合并为一条统一规则。

| 替换前 | 替换后 |
|--------|--------|
| 模式A（警情分析）+ 模式B（专项分析）+ 模式C（对策建议） | 统一逻辑链：数据呈现→特征归纳→风险研判→对策建议 |
| 分别描述每种模式的适用章节 | 一条规则覆盖所有章节 |

**适用条件**：多种模式之间的差异仅在深度和侧重点上，核心结构一致。
**不适用条件**：模式之间有本质性的结构差异时，应保留区分。

## 应用优先级

执行分析时，按以下顺序应用策略：

1. **先判断稳定性**（策略三）→ 决定哪些该具体、哪些该抽象
2. **对不稳定的内容做参数化**（策略一）→ 替换具体数据为占位符
3. **对穷举列表做规则提取**（策略二）→ 压缩为生成规则
4. **检查组织视角**（策略四）→ 从"有什么"转为"需要什么"
5. **检查是否可归一化**（策略五）→ 合并本质相同的模式
