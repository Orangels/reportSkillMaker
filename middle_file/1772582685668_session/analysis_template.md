# 模板分析报告

## 一、文档基本信息

- **文档类型**：统计分析报告（混合型：数据统计 + 分析研判 + 工作建议）
- **文档目的**：对辖区月度警情进行统计分析、趋势研判，并提出工作建议
- **发文单位**：临高县公安局情报指挥中心
- **受众**：各所、队、室（中心）；抄报上级领导
- **报告周期**：月度

---

## 二、报告结构框架

### 固定结构体系（五大模块）

报告采用 **"红头文件"** 格式，整体为 **总-分-总** 结构：

```
[红头区域] 发文单位名称 + 红色分隔线
[主标题]   "关于{X}月份警情分析的报告"

一、整体情况                    ← 总览段（固定）
二、上升警情类别分布            ← 上升警情分析（动态章节数）
    （一）{警情类型A}分析
        1.从{维度1}分析。
        2.从{维度2}分析。
        ...
        N.从辖区分布分析。      ← 辖区分布通常为最后一个分析维度
        小结：                  ← 每个二级章节末尾有小结
    （二）{警情类型B}分析
        ...
    （三）{特定关注区域/专项}分析  ← 可选，如"金牌港重点园区"
三、下降警情类型分布            ← 下降警情分析（动态章节数）
    （一）{警情类型C}分析
        1.从{维度1}分析。
        ...
        小结：
四、工作建议                    ← 工作建议（动态条数）
    （一）{建议标题}。{具体措施}
    （二）{建议标题}。{具体措施}
    ...

[落款]   发文单位 + 日期
[页尾]   抄送 + 抄报 + 印发信息
```

### 标题层级体系

| 层级 | 格式 | 示例 |
|------|------|------|
| 一级标题 | "一、二、三、四、" | 一、整体情况 |
| 二级标题 | "（一）（二）（三）" | （一）治安殴打他人警情分析 |
| 三级标题 | "1. 2. 3." | 1.从发案区域分析。 |
| 四级标题 | "（1）（2）（3）" | （1）殴打警情。 |
| 小结标记 | "小结：" | 小结：治安殴打他人警情... |
| 顺序词 | "一是 二是 三是" | 一是...；二是...；三是... |

### 内容组织规则

- **二、三章的章节划分规则**：按环比变化方向（上升/下降/持平）对警情大类进行分组
  - 第二章收录环比上升的警情类型
  - 第三章收录环比下降的警情类型
  - 可能存在"持平"类型，按实际情况归入
- **每个二级章节的分析维度数量**：根据该类警情的数据特征选择 3-5 个分析维度，不固定
- **分析维度的选取规则**：从数据特征出发，常见维度包括：发案区域、发生原因、行为手段、警情类型、高发类型、敏感警情、发案时间、辖区分布等。辖区分布通常作为最后一个分析维度
- **辖区分布可细分**：当某类警情包含子类型时，辖区分布可拆分为多个四级条目分别统计
- **特定关注区域/专项**：可作为单独的二级章节出现（如"金牌港重点园区"），分析结构与普通警情类型类似
- **工作建议的组织规则**：每条建议对应前述分析中的一个或多个突出问题，建议标题为概括性短语，正文为具体措施

### 3.2 段落格式

- **全文行距**：固定值 560 twips（即28磅），lineRule="exact"
- **首行缩进**：
  - 标准正文和标题：firstLine=640, firstLineChars=200（2个字符）
  - 部分段落：firstLine=643, firstLineChars=200
- **对齐方式**：正文 jc="both"（两端对齐），主标题 jc="center"（居中），落款偏右缩进
- **其他段落属性**：widowControl=0, kinsoku, overflowPunct, snapToGrid

### 3.3 页面设置

- **纸张**：A4 (w=11906, h=16838 twips)
- **页边距**：上1962, 下1848, 左1587, 右1474 twips
- **页眉/页脚**：header=851, footer=992 twips
- **页码格式**：十进制(decimal)
- **文档网格**：lines类型, linePitch=312

### 3.4 特殊格式元素

- **红头区域**：发文单位名居中，下方有红色分隔线（thinThick 样式，宽度57150 EMU，红色FF0000）
- **页尾区域**：抄送/抄报行上下各有黑色实线分隔（宽度12700和9525 EMU，黑色000000）
- **空白段落**：落款与页尾之间有多个空白段落用于分页留白

## 四、内容逻辑分析

### 4.1 整体情况段（第一章）

**固定句式模板**：
```
{统计起始日期}至{统计截止日期}我局共接报有效警情{总数}起（不含骚扰警情{骚扰数}起），
环比{升降}{百分比}%。其中{警情类型1}{数量}起，环比{升降}{百分比}%；
{警情类型2}{数量}起，环比{升降}{百分比}%；...
```

**数据指标**：
- 有效警情总数 + 环比变化率
- 骚扰警情数（排除项）
- 各大类警情数 + 环比变化率
- 大类分类体系：刑事警情、治安警情、交通警情、纠纷警情、群众紧急求助、其他警情

**加粗规则**：
- 警情类型名称加粗（如"有效警情"、"刑事警情"、"治安警情"等）
- 数字和环比数据不加粗
- "不含骚扰警情"加粗

### 4.2 上升/下降警情分析（第二、三章）

**每个警情类型的分析模块结构**：

1. **总量概述**（一句话）：我局共接报{警情类型}{数量}起，环比{升降}{百分比}%。
2. **多维度分析**（N个三级条目，每条结构为）：
   - 标题加粗：`N.从{分析维度}分析。`
   - 正文不加粗：`主要集中在{类别1}{数量}起（占比{百分比}%），其次{类别2}{数量}起（占比{百分比}%）...`
3. **小结**：
   - "小结："加粗
   - 正文不加粗，总结该类警情特征、趋势和简要建议
   - 使用"一是...；二是...；三是..."结构分点阐述

**分析维度的常见类型**：
- 发案区域/地点分析
- 发生原因分析
- 行为手段分析
- 警情类别/类型分析
- 高发类型分析
- 敏感警情分析
- 涉刀/涉特定要素分析
- 发案时间分析
- 辖区分布分析（可含子分类）

**数据表达模式**：
- 排名法："主要集中在A{数量}起（占比{百分比}%），其次B{数量}起（占比{百分比}%），C{数量}起（占比{百分比}%）"
- 列举法："{类别1}{数量}起，{类别2}{数量}起，{类别3}{数量}起"
- 环比法："{类别}{数量}起，环比{升降}{百分比}%"
- 混合法：同一段落中可混合使用以上多种表达

### 4.3 工作建议（第四章）

**每条建议的结构**：
- 二级标题用楷体：`（N）{概括性标题短语}。`
- 正文用仿宋：具体措施和要求
- 建议内容对应前述分析中发现的突出问题
- 措施具体，包含：责任主体、行动内容、频次要求、效果目标

### 4.4 数据计算规则

| 计算项 | 公式 | 说明 |
|--------|------|------|
| 环比变化率 | (本月-上月)/上月 * 100% | 保留一位小数 |
| 占比 | 该类数量/该维度总量 * 100% | 保留一位小数 |
| 有效警情 | 总警情 - 骚扰警情 | 明确标注排除项 |
| 各类警情之和 | 应等于有效警情总数 | 总分验证 |

## 五、数据/信息关联关系

### 5.1 总分验证关系

- 有效警情总数 = 刑事 + 治安 + 交通 + 纠纷 + 紧急求助 + 其他
- 每个维度分析中各子类之和应等于该类警情总数（或接近，允许合理误差）
- 占比之和应约等于100%（因四舍五入可能不精确等于100%）

### 5.2 环比一致性

- 整体情况中各类环比方向必须与第二、三章的归类一致
- 环比上升的类型应出现在第二章
- 环比下降的类型应出现在第三章

### 5.3 数据流向

```
原始警情数据
  ├─ 按类型分类 → 整体情况段
  ├─ 按环比升降 → 二/三章归类
  ├─ 按分析维度 → 各三级条目
  └─ 分析结论 → 四章工作建议
```

## 六、表达方式和语言风格

### 6.1 语言风格

- **正式程度**：高度正式，公文体
- **客观性**：数据描述客观准确，分析研判部分允许适度主观判断
- **专业术语**：大量使用公安行业术语（接报、辖区、环比、见警率、管事率、快侦快办等）

### 6.2 句式特征

- **数据描述句**："{主体}共接报{类型}{数量}起，环比{升降}{百分比}%"
- **排序描述句**："主要集中在{A}{数量}起（占比{百分比}%），其次{B}{数量}起（占比{百分比}%）"
- **分点阐述句**："此类警情特征突出：一是{要点1}；二是{要点2}；三是{要点3}"
- **建议措施句**："{责任主体}要{动作}，{目标}，{效果}"

### 6.3 常用连接词和表达

- 趋势描述："呈现明显反弹态势"、"环比均呈下降态势"、"管控成效显著"
- 程度描述："大幅上升"、"小幅上升"、"显著上升"、"明显增长"
- 建议措施："切实"、"全力"、"持续强化"、"深化"、"全覆盖"、"常态化"
- 因果连接："因...，..."、"反映出...需进一步..."

## 七、可变元素分析

### 7.1 固定元素（每期不变）

- 发文单位名称："临高县公安局情报指挥中心"
- 整体结构框架：红头 + 四章 + 落款 + 页尾
- 大类警情分类体系：刑事、治安、交通、纠纷、紧急求助、其他
- 格式规范：字体、字号、行距、缩进等
- 句式模板和公文用语风格
- 抄送/抄报对象格式

### 7.2 可变元素（每期随数据变化）

- **时间信息**：月份、统计起止日期、报告日期
- **数据数值**：各类警情数量、环比变化率、占比
- **上期数据**：用于计算环比
- **具体排名和分布**：各维度内的具体排序

### 7.3 动态元素（结构可变）

- **第二章/第三章的二级章节数量**：由当月哪些类型上升/下降决定，不固定
- **每个二级章节的分析维度数量**：根据该类警情数据特征选择3-5个，不固定
- **分析维度的具体名称**：根据该类警情特点选择，不是所有类型都分析相同维度
- **辖区分布下的四级条目**：视是否需要细分子类型而定
- **工作建议的条数和内容**：由前述分析中发现的问题决定
- **特殊专项章节**：如"金牌港重点园区"等，视当月是否有需要特别关注的区域/专项而定
- **小结中"一是二是三是"的条数**：根据特征数量动态调整

## 八、数据提取清单

### 8.1 整体情况数据（必需）

| 序号 | 数据项 | 数据类型 | 说明 |
|------|--------|---------|------|
| 1 | 统计月份（本期） | 月份 | 报告分析的目标月份 |
| 2 | 统计起止日期 | 日期范围 | 本期的具体起止日期（如"10月1日至30日"） |
| 3 | 有效警情总数（本期） | 数量 | 不含骚扰警情 |
| 4 | 有效警情总数（上期） | 数量 | 用于计算环比 |
| 5 | 骚扰警情数（本期） | 数量 | 排除项 |
| 6 | 各大类警情数量（本期） | 数量*6 | 刑事、治安、交通、纠纷、紧急求助、其他 |
| 7 | 各大类警情数量（上期） | 数量*6 | 用于计算环比 |

### 8.2 上升警情详细分析数据

对于每个环比上升的警情大类或关注类型，需提取以下多维度数据：

| 序号 | 数据维度 | 数据内容 | 说明 |
|------|---------|---------|------|
| 1 | 该类警情总量 | 本期数量 + 上期数量 | 用于计算环比 |
| 2 | 发案区域/地点分布 | 各区域类别 + 数量 | 按数量降序，带占比 |
| 3 | 发生原因分布 | 各原因类别 + 数量 | 按数量降序，带占比 |
| 4 | 行为手段分布 | 各手段类别 + 数量 | 按数量降序，带占比；涉刀需列明刀具类型 |
| 5 | 警情类型/子类分布 | 各子类别 + 数量 | 按数量降序，带占比 |
| 6 | 高发类型分布 | 高发具体类别 + 数量 | 按数量降序 |
| 7 | 敏感警情明细 | 具体案件类型 + 数量 | 性质恶劣的特殊案件 |
| 8 | 涉特定要素分析 | 如涉刀警情本期+上期 | 含案件处理结果、伤亡情况 |
| 9 | 辖区分布 | 各派出所 + 数量 | 按数量降序，带占比 |
| 10 | 辖区分布-子类型 | 各子类型按辖区分布 | 当需要细分时 |

**注意**：并非每个警情类型都需提取以上全部维度。根据实际数据中可用的字段和分类，选择能支撑分析的维度。

### 8.3 下降警情详细分析数据

与上升警情类似的多维度数据结构，但分析重点不同：

| 序号 | 数据维度 | 数据内容 | 说明 |
|------|---------|---------|------|
| 1 | 该类警情总量 | 本期数量 + 上期数量 | 用于计算环比 |
| 2 | 警情类别分布 | 各子类别 + 本期数量 + 上期数量 | 带环比 |
| 3 | 发案时间分布 | 各时段 + 数量 | 可按小时段/日期类型统计 |
| 4 | 周末/节假日数据 | 周末事故数 + 占比 | 与工作日对比 |
| 5 | 辖区分布 | 各辖区 + 数量 | 按数量降序 |

### 8.4 特定关注区域/专项数据

| 序号 | 数据维度 | 数据内容 | 说明 |
|------|---------|---------|------|
| 1 | 该区域/专项警情总量 | 本期 + 上期 | 环比 |
| 2 | 按警情大类分布 | 各类别 + 本期 + 上期 | 带环比 |
| 3 | 重点类型细分 | 如治安细分、纠纷细分 | 本期+上期，带环比 |

### 8.5 辖区基础信息

| 序号 | 数据项 | 说明 |
|------|--------|------|
| 1 | 辖区派出所列表 | 所有派出所名称 |
| 2 | 重点关注区域 | 如"金牌港重点园区"等 |

### 8.6 工作建议所需关联数据

工作建议不需要额外提取数据，由前述分析数据中的突出问题自动推导。Writer 需根据分析结论撰写针对性建议。

## 九、验证检查清单

### 数据验证

- [ ] 各大类警情数量之和 = 有效警情总数
- [ ] 环比计算正确（(本期-上期)/上期 * 100%，保留一位小数）
- [ ] 各维度分析中子项之和 = 该类总数（允许合理误差）
- [ ] 占比计算正确且之和约等于100%
- [ ] 上升类型归入第二章，下降类型归入第三章

### 结构验证

- [ ] 四大章节完整（整体情况、上升警情、下降警情、工作建议）
- [ ] 每个二级章节包含：总量概述 + 多维度分析 + 小结
- [ ] 工作建议条数与分析中的突出问题对应

### 格式验证

- [ ] 红头区域格式正确（方正小标宋简体、55磅、红色、红色分隔线）
- [ ] 主标题格式正确（方正小标宋简体、22磅、居中）
- [ ] 一级标题用黑体16磅
