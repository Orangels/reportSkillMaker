## 一、文档基本信息

| 属性 | 内容 |
|------|------|

| 文档目的 | 对某月份辖区警情进行全面统计分析、趋势研判和提出工作建议 |
| 发文单位 | 临高县公安局情报指挥中心 |

## 二、报告结构框架

### 整体结构（固定框架）

报告采用"总-分-建议"三段式结构：

```
[红头] 发文单位名称 + 红色分隔线
[标题] 关于X月份警情分析的报告
[正文]
  一、整体情况（总述段）
  二、上升警情类别分布（分述：上升类）
    （一）[上升警情类型A]警情分析
      概述段 → 多维度分析段(1.从...分析) → 小结段
    （二）[上升警情类型B]警情分析
      概述段 → 多维度分析段 → 小结段
    ...（数量根据实际上升的警情类型确定）
  三、下降警情类型分布（分述：下降类）
    （一）[下降警情类型A]警情分析
      概述段 → 多维度分析段 → 小结段
    ...（数量根据实际下降的警情类型确定）
  四、工作建议
    （一）[建议标题A]。[建议正文]
    （二）[建议标题B]。[建议正文]
    ...（每条建议与前面分析的警情类型对应）
[署名] 发文单位 + 日期
[分隔线]
[抄送/抄报] 抄送对象 + 抄报对象
[印发行] 印发单位 + 印发日期
```

### 内容组织规则

1. **一级章节划分规则**：固定为"整体情况"、"上升警情类别分布"、"下降警情类型分布"、"工作建议"四个部分
2. **二级章节归属规则**：根据环比升降，将警情类型分别归入"上升"或"下降"章节
3. **二级章节数量规则**：上升/下降警情的子章节数量根据实际数据中环比上升/下降的重点警情类型确定，不固定
4. **三级分析维度规则**：每个警情类型的多维度分析条目（"从...分析"）根据该类型的特点和可用数据确定，不固定，常见维度包括：发案区域、发生原因、行为手段、警情类型、高发类型、敏感警情、发案时间、辖区分布等
5. **四级子维度规则**：在"辖区分布"等维度下，可能存在进一步的子分类（如(1)XX警情、(2)YY警情），根据数据特点确定
6. **工作建议对应规则**：每条工作建议与前面分析的某个或某几个警情类型对应，建议数量与分析的主要问题类型数量大致匹配
7. **排序规则**：各类型内的具体数据按数量降序排列，使用"主要集中在...，其次..."句式

## 四、内容逻辑分析

### 4.1 整体情况段（总述）

**组织逻辑**：
- 先报告统计时间范围和有效警情总量（不含骚扰警情），附环比变化
- 然后按警情大类逐一列出数量和环比变化
- 警情大类名称加粗，数量和百分比不加粗

**句式模板**：
```
[上月]X日至X日我局共接报**有效警情**N起（**不含骚扰警情**M起），环比[上升/下降]X%。
其中**[类型A]**N起，环比[上升/下降]X%；**[类型B]**N起，环比[上升/下降]X%；...。
```

**固定的警情大类**（需要从数据中获取各自的数量和环比）：
- 刑事警情、治安警情、交通警情、纠纷警情、群众紧急求助、其他警情

### 4.2 上升/下降警情分析段（分述）

**归类逻辑**：根据环比结果将值得重点分析的警情类型归入"上升"或"下降"章节

**每个警情类型的分析结构**：
1. **概述段**：报告该类型警情总量和环比变化
   - 句式：`我局共接报**[警情类型]**N起，环比[上升/下降]X%。`
2. **多维度分析段**：根据该类型的特点，从多个维度进行分析
   - 每个维度的分析标题加粗，格式为`N.从[维度名称]分析。`
   - 正文使用"主要集中在...N起（占比X%），其次...N起（占比X%）"的句式
   - 按数量降序排列
   - 如有需要，可在某个维度下设置子分类，格式为`（N）[子类名]。`
3. **小结段**：对该类型警情进行综合评价和趋势判断
   - "小结："加粗
   - 使用"一是...二是...三是..."分点总结的结构（加粗序号词）
   - 包含趋势判断 + 特征总结 + 建议方向

### 4.3 工作建议段

**组织逻辑**：
- 每条建议用"（N）"编号，标题为楷体
- 标题后紧跟句号，然后是建议正文（仿宋）
- 建议内容结构：针对问题 → 责任主体 → 具体措施 → 量化要求 → 预期效果
- 建议与前面分析的警情类型一一对应或关联

**句式特征**：
- 使用指令性语言："要"、"需"、"应"、"切实"
- 包含具体责任主体："各派出所"、"XX所"、"交警部门"
- 包含量化指标："每月不少于N次"、"每周不少于N次"
- 包含结果导向表述："切实压降"、"有效遏制"、"全力守护"

## 五、数据/信息关联关系

### 5.1 总分关系

```
有效警情总量 = 刑事警情 + 治安警情 + 交通警情 + 纠纷警情 + 群众紧急求助 + 其他警情
  （注意：有效警情不含骚扰警情）
```

### 5.2 环比计算公式

```
环比变化率 = (本月数量 - 上月数量) / 上月数量 * 100%
- 正值为"上升"
- 负值为"下降"
- 零值为"持平"
```

### 5.3 占比计算

```
占比 = 某细分类数量 / 该维度总数量 * 100%
```

### 5.4 验证规则

1. 各警情大类数量之和应等于有效警情总量
2. 同一维度下各细分类的占比之和应接近100%
3. 环比变化方向（上升/下降）应与数值正负一致
4. 本月数量和环比变化率应能反推出上月数量
5. "上升警情"章节中的所有类型环比应为正值
6. "下降警情"章节中的所有类型环比应为负值或接近持平

## 六、表达方式和语言风格

### 6.1 语言风格

| 特征 | 说明 |
|------|------|
| 正式程度 | 高度正式的公文语言 |
| 客观性 | 数据驱动，先数据后判断 |
| 专业性 | 使用公安系统专业术语 |
| 语气 | 客观陈述 + 工作指导性 |

### 6.2 专业术语

- 警情类型：有效警情、骚扰警情、刑事警情、治安警情、交通警情、纠纷警情、紧急求助、涉刀警情、涉未成人警情
- 统计术语：环比、占比、接报、发案
- 区域术语：辖区、派出所（XX所）、镇村
- 行动术语：严打整治、护苗行动、晚安行动、亮灯巡逻、快侦快办
- 管理术语：见警率、管事率、重点人员台账、矛盾排查、风险防范

### 6.3 句式结构特点

1. **数据陈述句式**：
   - `我局共接报**[类型]**N起，环比[上升/下降]X%。`
   - `主要集中在[类别]N起（占比X%），其次[类别]N起（占比X%）。`
   - `较上月N起环比[上升/下降]X%。`

2. **分析归纳句式**：
   - `此类警情特征突出：**一是**...；**二是**...；**三是**...。`
   - `[类型]呈现明显[上升/下降]态势，且[升降]幅度较大。`

3. **建议措施句式**：
   - `[责任主体]要[具体措施]，每[周/月]不少于N次，[预期效果]。`
   - `切实[动词]+[目标]，有效遏制此类警情[反弹/高发]势头。`

4. **加粗规则**：
   - 警情类型名称在概述段首次出现时加粗
   - 分析维度标题（"N.从...分析。"）加粗
   - "小结："加粗
   - 分点序号词（"一是"、"二是"）加粗
   - 行内警情子类标题加粗（如"从治安警情分析，"）

## 九、验证检查清单

### 数据准确性验证
- [ ] 各警情大类数量之和等于有效警情总量
- [ ] 有效警情 + 骚扰警情 = 总接报数
- [ ] 所有环比变化率计算正确（(本月-上月)/上月*100%）
- [ ] 同一维度内各项占比之和接近100%
- [ ] "上升警情"章节中所有类型的环比确实为正值
- [ ] "下降警情"章节中所有类型的环比确实为负值

### 内容完整性验证
- [ ] 整体情况段包含了所有警情大类
- [ ] 每个重点分析的警情类型都有概述段、多维度分析段和小结段
- [ ] 多维度分析覆盖了数据源中可用的主要维度
- [ ] 每条工作建议都对应了某个分析发现的问题
- [ ] 抄送、抄报、印发信息完整
